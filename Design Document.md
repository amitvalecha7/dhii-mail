# 1. Design Philosophy

## Core Principles

| Principle | Description |
|-----------|-------------|
| **Chat-First** | Every action can be accomplished via natural language |
| **Context-Aware** | UI remembers user preferences, rules, and history |
| **Progressive Disclosure** | Complexity reveals itself only when needed |
| **Unified Canvas** | Everything renders in one place â€” no modals, no page switches |
| **Stream-Native** | UI builds progressively, never blocks |

```json
{
  "type": "importance_grid",
  "algorithm": "masonry",
  "components": [
    {
      "component_id": "crm-pipeline",
      "component_type": "stat_card",
      "importance": 0.9,
      "props": { "title": "Pipeline", "value": "$1.2M" }
    },
    {
      "component_id": "email-summary",

      "importance": 0.3,
      "props": { "title": "Unread Emails", "items": [...] }
    }
  ],
  "responsive": {
    "desktop": { "columns": 3 },
    "tablet": { "columns": 2 },
    "mobile": { "columns": 1 }
  }
}
```

**Behavior**:
- Components with higher `importance` scores (0.0-1.0) receive:
  - **Larger grid areas** (e.g., `grid-column: span 2`)
  - **Higher visual priority** (positioned top-left)
  - **Richer styling** (larger fonts, more prominent borders)
- Algorithm uses CSS Grid `grid-template-areas` to dynamically allocate space.
- Example: `importance: 0.9` â†’ 2x2 grid area, `importance: 0.3` â†’ 1x1 grid area.

## 9.2 Stream Handling

```typescript
// Pseudo-code for stream handling
async function handleStream(prompt: string) {
  // 1. Clear canvas, show "thinking"
  dispatch({ type: 'CANVAS_START_STREAM', prompt });
  
  // 2. Open SSE connection
  const stream = new EventSource(`/api/stream?prompt=${encodeURIComponent(prompt)}`);
  
  stream.onmessage = (event) => {
    const chunk = JSON.parse(event.data);
    
    switch (chunk.chunk_type) {
      case 'skeleton':
        dispatch({ type: 'CANVAS_ADD_SKELETON', chunk });
        break;
      
      case 'display':
        dispatch({ type: 'CANVAS_HYDRATE_COMPONENT', chunk });
        break;
      
      case 'config':
        dispatch({ type: 'SETTINGS_UPDATE', chunk });
        dispatch({ type: 'CANVAS_ADD_COMPONENT', chunk.display });
        break;
      
      case 'rule':
        dispatch({ type: 'RULES_UPSERT', chunk });
        dispatch({ type: 'CANVAS_ADD_COMPONENT', chunk.display });
        break;
      
      case 'clarification':
        dispatch({ type: 'CANVAS_PAUSE_FOR_CLARIFICATION', chunk });
        break;
      
      case 'error':
        dispatch({ type: 'CANVAS_HANDLE_ERROR', chunk });
        break;
    }
  };
  
  stream.onerror = () => {
    dispatch({ type: 'CANVAS_STREAM_ERROR' });
    stream.close();
  };
}
```

**Behavior**:

3. **Cache Invalidation**: 
   - After `cache_ttl_minutes` (default: 15 minutes)
   - When user manually refreshes
   - When underlying data changes (via webhooks)

**Purpose**: Provide a centralized, universal view of all plugins in the system, allowing users to discover, install, configure, and manage plugins from a single interface.

**Philosophy**: Similar to VS Code Extensions, Chrome Web Store, or Shopify App Store - a marketplace feel with transparency and ease of use.

        async for chunk in orchestrator.process(prompt):
